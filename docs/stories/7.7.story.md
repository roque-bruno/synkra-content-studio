# Story 7.7 — QA End-to-End — Fluxo Completo de Configuracao

**Epico:** 7 — Configuracao Self-Service
**Status:** Ready
**Executor:** @qa (Quinn)
**Modo:** YOLO
**Dependencia:** 7.5, 7.6

---

## Descricao

Teste end-to-end do fluxo completo: gestora abre Settings, configura API key, testa conexao, salva, e verifica que o status reflete em toda a plataforma. Validar que o objetivo do Epic 7 foi alcancado: autonomia total sem depender do squad de infra.

## Acceptance Criteria

- [ ] Fluxo completo funciona: abrir Settings → inserir key → testar → salvar → status atualizado
- [ ] Status de plataforma muda de "Nao configurado" para "Conectado" em TODAS as paginas
- [ ] Remocao de key funciona: remover → status volta para "Nao configurado"
- [ ] Key invalida: teste falha → erro exibido → key NAO salva
- [ ] Restart do container: keys persistem e status mantem
- [ ] Preview mode: sem Google API key → banner de preview aparece
- [ ] Com Google API key: banner de preview desaparece
- [ ] Publicacao: so plataformas conectadas disponiveis
- [ ] Relatorio de QA gerado

## Scope

**IN:**
- Teste E2E de todo o fluxo de configuracao
- Teste de persistencia (restart container)
- Teste de estados cruzados (settings → dashboard → publicar)
- Regressao de funcionalidades existentes

**OUT:**
- Testes automatizados (fora do escopo deste epic)
- Teste de carga/stress

## Cenarios de Teste

### Cenario 1: Primeiro uso (fresh install)
1. Abrir Settings
2. Todas as integracoes: "Nao configurado"
3. Dashboard: "0 de 5 integracoes ativas"
4. Publicar: botao desabilitado
5. Banner de preview mode visivel

### Cenario 2: Configurar Google AI
1. Inserir GOOGLE_API_KEY valida
2. Clicar "Testar Conexao" → "Conexao OK"
3. Clicar "Salvar" → checkmark verde
4. Banner de preview mode desaparece
5. Dashboard: "1 de 5 integracoes ativas"

### Cenario 3: Key invalida
1. Inserir key invalida
2. Clicar "Salvar" → teste automatico → "Falha: API key invalida"
3. Key NAO salva, campo continua editavel

### Cenario 4: Remover key
1. Clicar "Remover" em Google AI
2. Confirmacao: "Tem certeza?"
3. Status volta para "Nao configurado"
4. Banner de preview mode reaparece

### Cenario 5: Persistencia
1. Configurar key
2. Restart container (docker compose restart)
3. Abrir Settings → key ainda configurada (mascarada)

## Tasks

1. Executar cenario 1 (fresh install)
2. Executar cenario 2 (configurar Google AI)
3. Executar cenario 3 (key invalida)
4. Executar cenario 4 (remover key)
5. Executar cenario 5 (persistencia)
6. Verificar regressao de funcionalidades existentes
7. Gerar relatorio de QA
