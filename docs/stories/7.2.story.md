## Descricao

Criar endpoints REST para que o frontend consiga ler e salvar API keys e configuracoes de integracao. Implementar conforme ADR definido na Story 7.1.

## Acceptance Criteria

- [ ] `GET /api/settings/integrations` — retorna lista de integracoes com status e keys mascaradas
- [ ] `PUT /api/settings/integrations/{platform}` — salva/atualiza credenciais de uma plataforma
- [ ] `DELETE /api/settings/integrations/{platform}` — remove credenciais de uma plataforma
- [ ] Keys retornadas ao frontend SEMPRE mascaradas (ex: `****last4`)
- [ ] Keys salvas conforme modelo de storage do ADR (Story 7.1)
- [ ] Endpoints protegidos por autenticacao
- [ ] Validacao de input: rejeitar keys vazias ou com formato invalido
- [ ] Persistencia sobrevive restart do container (volume Docker)

## Scope

**IN:**
- 3 endpoints REST (GET, PUT, DELETE)
- Armazenamento seguro conforme ADR
- Mascaramento de keys na resposta
- Auth obrigatoria em todos os endpoints

**OUT:**
- Validacao real das keys (Story 7.4)
- Frontend (Story 7.3)

## API Design

### GET /api/settings/integrations
```json
{
  "integrations": {
    "google_ai": {
      "configured": true,
      "key_masked": "****abc1",
      "last_tested": "2026-03-31T10:00:00Z",
      "status": "valid"
    },
    "instagram": {
      "configured": false,
      "key_masked": null,
      "last_tested": null,
      "status": "not_configured"
    }
  }
}
```

### PUT /api/settings/integrations/{platform}
```json
{
  "api_key": "actual-key-value",
  "extra": {
    "page_id": "123456",
    "org_id": "abcdef"
  }
}
```

### Response
```json
{
  "ok": true,
  "platform": "google_ai",
  "status": "saved",
  "key_masked": "****abc1"
}
```
