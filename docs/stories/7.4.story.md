# Story 7.4 — Validacao Real de API Keys via Chamada Externa

## Descricao

Integrar o teste real de conexao (endpoint da Story 6.4) com o fluxo de configuracao pela UI. Quando a gestora salva uma API key, o sistema deve automaticamente testar a conexao e reportar o resultado.

## Acceptance Criteria

- [ ] Botao "Testar Conexao" por plataforma na pagina de Settings
- [ ] Teste dispara chamada ao endpoint `/api/settings/test/{platform}`
- [ ] Resultado exibido inline: verde "Conexao OK (XXms)" ou vermelho "Falha: {erro}"
- [ ] Teste automatico ao salvar nova key (validar antes de persistir)
- [ ] Se validacao falha, key NAO e salva — exibir erro e manter campo editavel
- [ ] Se validacao passa, key e salva e status atualizado para "Conectado"
- [ ] Loading spinner durante teste (pode levar ate 10s)
- [ ] Teste manual disponivel a qualquer momento (nao so ao salvar)

## Scope

**IN:**
- Botao "Testar Conexao" no frontend
- Validacao automatica ao salvar
- Feedback visual de resultado do teste
- Integracao com endpoint de teste (Story 6.4)

**OUT:**
- Teste periodico automatico (health check cron)
- Notificacoes de expiracao de token

## Tasks

1. Adicionar botao "Testar Conexao" em cada card de plataforma
2. Implementar chamada ao `/api/settings/test/{platform}` com loading state
3. Exibir resultado inline (sucesso/erro/latencia)
4. Integrar validacao no fluxo de save: testar → se OK → salvar → atualizar status
5. Tratar timeout e erros de rede no frontend
