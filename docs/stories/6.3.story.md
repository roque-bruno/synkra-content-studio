# Story 6.3 — Desabilitar Publicacao sem Configuracao Valida

**Epico:** 6 — Transparencia de Producao
**Status:** Done
**Executor:** @dev (Dex)
**Modo:** YOLO
**Dependencia:** 6.2

## Descricao

O botao "Publicar" esta sempre habilitado e, quando clicado em preview mode, retorna IDs falsos (preview-xxxx) com toast de sucesso "Publicado com sucesso!". Isso e gravemente enganoso — a gestora pode acreditar que publicou quando nada aconteceu.

Desabilitar o botao de publicar quando nenhuma plataforma esta realmente conectada. Em preview mode, o toast deve indicar claramente que e uma simulacao.

## Acceptance Criteria

- [x] Botao "Publicar" desabilitado (grayed out) quando TODAS as plataformas estao `connected: false`
- [x] Tooltip no botao desabilitado: "Configure as plataformas em Settings para publicar"
- [x] Em preview mode, se publicacao for permitida por alguma razao, toast mostra "Simulacao (Preview)" em vez de "Publicado com sucesso!"
- [x] Seletores de plataforma no modal de publicacao mostram apenas plataformas conectadas
- [x] Backend retorna campo `preview: true` na resposta de publicacao em preview mode
- [x] Nenhum ID falso (preview-xxxx) e apresentado como se fosse real

## Scope

**IN:**
- Desabilitar botao Publicar baseado em status real das plataformas
- Diferenciar toast de preview vs producao
- Filtrar plataformas no modal de publicacao

**OUT:**
- Mudar logica de preview mode no backend (apenas sinalizar)
- Implementar publicacao real (ja existe, depende de API keys)

## Tasks

1. Adicionar propriedade `hasConnectedPlatforms` computada a partir de `publishStatus`
2. Condicionar botao Publicar a `hasConnectedPlatforms`
3. Adicionar tooltip explicativo no botao desabilitado
4. Modificar toast de publicacao para indicar preview mode
5. Filtrar plataformas desconectadas do modal de selecao
6. Backend: adicionar `preview: true` na resposta de `/api/publish`
