## Descricao

O backend atualmente verifica apenas `bool(api_key)` — se a variavel existe, considera "configurado". Nunca faz uma chamada real para validar se a credencial funciona. Implementar teste real de conexao para cada plataforma.

## Acceptance Criteria

- [x] Endpoint `/api/settings/test/{platform}` faz chamada real para API da plataforma
- [x] Google AI: `POST https://generativelanguage.googleapis.com/v1beta/models?key=X` — verifica resposta 200
- [x] Instagram: `GET https://graph.instagram.com/me?access_token=X` — verifica resposta 200
- [x] Facebook: `GET https://graph.facebook.com/me?access_token=X` — verifica resposta 200
- [x] LinkedIn: `GET https://api.linkedin.com/v2/me` com header Bearer — verifica resposta 200
- [x] YouTube: `GET https://www.googleapis.com/youtube/v3/channels?mine=true&key=X` — verifica resposta 200
- [x] Cada teste retorna `{ok: bool, error: str|null, latency_ms: int}`
- [x] Timeout de 10 segundos por teste
- [x] Endpoint protegido por autenticacao (mesmo auth do sistema)

## Scope

**IN:**
- Novo endpoint `/api/settings/test/{platform}`
- Chamadas HTTP reais para cada API
- Retorno estruturado com status, erro e latencia

**OUT:**
- Salvamento automatico de credenciais (isso e Epic 7)
- UI para disparar testes (isso e Epic 7, Story 7.4)
- Cache de resultado de teste

## Tasks

1. Criar endpoint `/api/settings/test/{platform}` com auth
2. Implementar teste Google AI (generativelanguage API)
3. Implementar teste Instagram (Graph API)
4. Implementar teste Facebook (Graph API)
5. Implementar teste LinkedIn (REST API v2)
6. Implementar teste YouTube (Data API v3)
7. Timeout de 10s e tratamento de erros de rede
8. Integrar com `/api/publish/status` — popular `connected` com resultado do teste
