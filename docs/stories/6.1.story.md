# Story 6.1 — Banner Persistente de Preview Mode

**Epico:** 6 — Transparencia de Producao
**Status:** Done
**Executor:** @dev (Dex)
**Modo:** YOLO

---

## Descricao

Quando o sistema opera sem GOOGLE_API_KEY (preview mode), nenhum indicador visual diferencia da producao real. A gestora pode acreditar que tudo esta funcional quando na verdade a geracao de conteudo IA esta desabilitada.

Adicionar um banner persistente no topo de TODAS as paginas quando `preview_mode === true`, impossibilitando confusao.

## Acceptance Criteria

- [x] Banner amarelo/warning visivel no topo quando `preview_mode` e `true`
- [x] Texto: "Modo Preview — Geracao de conteudo IA desabilitada. Configure a API key em Settings."
- [x] Link "Settings" navega para a aba de configuracoes
- [x] Banner nao aparece quando `preview_mode` e `false` (API key configurada)
- [x] Banner usa `x-if` (nao `x-show`) para nao ocupar espaco no DOM quando inativo
- [x] Responsivo — funciona em desktop e mobile

## Scope

**IN:**
- Banner HTML/CSS no layout principal
- Leitura do campo `preview_mode` do endpoint `/api/settings`
- Estilizacao warning (amarelo/laranja)

**OUT:**
- Mudancas no backend
- Logica de quando preview_mode e true (ja existe)

## Tasks

1. Adicionar propriedade `previewMode` no `studioApp()` populada via `/api/settings`
2. Criar banner HTML com `<template x-if="previewMode">`
3. Estilizar com cores de warning e link para Settings
4. Testar com e sem API key
