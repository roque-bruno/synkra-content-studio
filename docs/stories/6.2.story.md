# Story 6.2 — Status Real de Conexao das Plataformas

**Status:** Done
**Executor:** @dev (Dex)

## Descricao

Atualmente, `/api/publish/status` retorna apenas `configured: bool` baseado na existencia de variavel de ambiente. As plataformas (Instagram, Facebook, LinkedIn, YouTube) mostram "Conectado" sem nenhuma conta real integrada. Isso engana a gestora.

Corrigir para que o status reflita a realidade: so mostrar "Conectado" quando houver credenciais validas E conexao testada.

## Acceptance Criteria

- [x] Endpoint `/api/publish/status` retorna `connected: false` quando nao ha credenciais configuradas
- [x] Frontend mostra "Nao configurado" (cinza) em vez de "Conectado" (verde) para plataformas sem credenciais
- [x] Frontend mostra "Erro de conexao" (vermelho) se credenciais existem mas validacao falha
- [x] Frontend mostra "Conectado" (verde) SOMENTE com credenciais validas e teste real OK
- [x] Cada plataforma tem status independente
- [x] Estado visual usa icones/cores distintas para cada status

## Scope

**IN:**
- Refatorar endpoint `/api/publish/status` para retornar status granular por plataforma
- Refatorar frontend para consumir novo formato de resposta
- Tres estados visuais: nao_configurado (cinza), erro (vermelho), conectado (verde)

**OUT:**
- Validacao real via API (isso e Story 6.4)
- Configuracao de credenciais pela UI (isso e Epic 7)

## Tasks

1. Refatorar `/api/publish/status` — retornar `{platform: {configured: bool, connected: bool, error: str|null}}`
2. Atualizar frontend `publishStatus` para ler novo formato
3. Criar 3 estados visuais distintos com cores e icones
4. Remover texto hardcoded "Conectado" — substituir por estado dinamico
5. Testar com 0 plataformas configuradas (estado padrao do deploy)
