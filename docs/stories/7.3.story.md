# Story 7.3 — UI Editavel de Settings com Validacao Inline

**Epico:** 7 — Configuracao Self-Service

**Dependencia:** 7.2

## Descricao

Transformar a pagina de Settings de read-only para editavel. A gestora deve conseguir inserir, alterar e remover API keys diretamente no browser, com feedback visual imediato.

## Acceptance Criteria

- [ ] Cada integracao tem campo de input para API key (type=password por padrao)
- [ ] Botao "Mostrar/Ocultar" para toggle de visibilidade da key
- [ ] Botao "Salvar" por plataforma — chama `PUT /api/settings/integrations/{platform}`
- [ ] Botao "Remover" por plataforma — chama `DELETE` com confirmacao
- [ ] Campos extras quando necessarios (Page ID para Instagram, Org ID para LinkedIn)
- [ ] Feedback visual: loading spinner durante save, checkmark verde no sucesso, X vermelho no erro
- [ ] Key existente mostra valor mascarado (****last4) e placeholder "Insira nova key para substituir"
- [ ] Validacao client-side: campo nao pode ser vazio ao salvar
- [ ] Design consistente com o restante da interface (Tailwind/tema existente)

## Scope

**IN:**
- Formulario editavel para cada plataforma
- Inputs de credenciais com toggle de visibilidade
- Feedback visual de save/error
- Design responsivo

**OUT:**
- Validacao real da key (Story 7.4)
- Botao "Testar conexao" (Story 7.4)

## Tasks

1. Refatorar secao de Settings de read-only para formulario editavel
2. Criar card por plataforma com input + save + remove
3. Implementar toggle mostrar/ocultar key
4. Integrar com endpoints CRUD (Story 7.2)
5. Loading states e feedback visual
6. Campos extras (page_id, org_id, channel_id) condicionais por plataforma
7. Validacao client-side basica
