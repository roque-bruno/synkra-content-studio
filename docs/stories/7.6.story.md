# Story 7.6 — QA de Seguranca — API Keys Nunca Expostas no Frontend

**Epico:** 7 — Configuracao Self-Service
**Status:** Ready

## Descricao

Auditoria de seguranca focada em garantir que API keys nunca sejam expostas ao frontend de forma insegura. Verificar todos os endpoints, respostas HTTP, HTML source e Network tab.

## Acceptance Criteria

- [ ] NENHUM endpoint retorna API key completa no response body
- [ ] GET /api/settings/integrations retorna apenas keys mascaradas (****last4)
- [ ] Network tab nao mostra keys em nenhuma request/response
- [ ] HTML source nao contem keys hardcoded
- [ ] Console do browser nao loga keys
- [ ] PUT request envia key no body (nao em query params ou URL)
- [ ] Todos os endpoints de settings exigem autenticacao
- [ ] Acesso sem auth retorna 401/403
- [ ] Keys nao aparecem em logs do servidor

## Scope

**IN:**
- Auditoria de todos os endpoints de settings
- Verificacao de Network tab no browser
- Verificacao de HTML source
- Verificacao de logs do servidor
- Teste de acesso sem autenticacao

**OUT:**
- Pen testing completo
- Auditoria de criptografia at-rest
- Compliance audit (LGPD, etc.)

## Checklist de Seguranca

### Endpoints
- [ ] GET /api/settings/integrations — keys mascaradas
- [ ] PUT /api/settings/integrations/{platform} — key no body, nao na URL
- [ ] DELETE /api/settings/integrations/{platform} — sem key na resposta
- [ ] GET /api/settings/test/{platform} — sem key na resposta
- [ ] GET /api/publish/status — sem keys

### Frontend
- [ ] HTML source: nenhuma key em atributos ou scripts
- [ ] Alpine.js data: keys nao armazenadas em x-data
- [ ] Console: nenhum console.log com keys

### Server
- [ ] Logs: nenhuma key em output de log
- [ ] Error messages: nenhuma key em stacktraces

## Tasks

1. Auditar respostas de todos os endpoints de settings
2. Verificar Network tab em todas as interacoes de settings
3. Inspecionar HTML source
4. Verificar logs do servidor durante operacoes de settings
5. Testar acesso sem autenticacao
6. Gerar relatorio de seguranca
